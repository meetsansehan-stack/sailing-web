# ICP 서베이 (구글폼) — 예비 초1(2020년생) 부모 · 심화판

> **용도**: [[ICP_INTERVIEW]]의 정량·넓이 버전. 객관식이라 응답률↑(오픈형은 거의 입력 안 함 — 실측). **인터뷰를 대체하지 않음** — 넓게 돌려 *핫 ICP를 발굴*하고 그 소수를 1:1로.
> **목표 단계 = need 검증**(가격 최적화 아님). 그래서 *과거 행동 > 진술 의향*으로 가중. 진술형(의향·WTP)은 over-claim 보정 후 2차 신호로만.

## 0. 방법론 요약 (이 설문이 따르는 전문 결정 7가지)

1. **행동 우선, 가설 후순위**: 서베이는 본질적으로 "~하실래요?"(가설)를 묻기 쉬움 → 의도적으로 **과거 시제·구체 회상**("지난 1년에 ~한 적, 몇 번")을 핵심에 배치. 진술 의향(Q14·Q16·WTP)은 *부풀려진다고 가정*하고 행동 문항으로 교차검증.
2. **솔루션 늦게**: 우리 컨셉은 **§F에서 처음** 노출. 앞 문항이 컨셉에 프라이밍되지 않게(The Mom Test의 서베이판).
3. **부정 보기 필수**: 모든 핵심 문항에 "걱정 안 됨 / 안 함 / 다 무료여야"를 넣음. 없으면 설문이 *내 가설만 확인*. 비타민이면 비타민으로 드러나야 함.
4. **편향 통제**: 무기명·"정답 없음" 명시(사회적 바람직성↓) / 보기 순서 랜덤화(묵종·순서 편향↓, 단 척도형·"기타"·"없음"은 고정) / 이중질문 금지(한 문항 한 개념) / loaded 워딩 제거.
5. **척도 규율**: 빈도는 **시간 앵커**("지난 1년", "최근 30일")로 구체화 / 만족·강도는 **균형 잡힌 라벨 척도**(중립점 포함) / 단일 vs 복수 명시.
6. **완료율 엔지니어링**: 모바일 퍼스트(짧은 문장)·진행률 표시·**분기**(맞벌이/놓침경험자만 추가 문항)로 1인당 체감 문항 축소. 목표 **3~4분·이탈 30%↓**. 어뷰징 대비 **주의력 체크 1개**.
7. **사전 분석 플랜**: §5에 각 문항→신호 매핑 + kill/scale 임계를 *데이터 보기 전에* 고정.

> ⚠️ **만나이 함정(2023 개정)**: "7세"는 세는나이/만나이 혼용으로 오스크린됨. 입학 아동 = **2020년생**(만 6세 되는 2026년의 다음 해, 2027.3 입학). 스크린은 나이 숫자 아닌 **출생연도+입학시점** 불변 앵커로.

## 3. 작성 크래프트 — 이 설문에 적용한 규칙 (확장 시 준수)

- **한 문항 한 개념**(이중질문 금지): "정보가 정확하고 빠른가?" ❌ → 정확/속도 분리.
- **균형 척도**: 긍정·부정 보기 수 대칭 + 중립점 1개. 긍정만 나열(=묵종 유도) 금지.
- **시간 앵커 빈도**: "자주/가끔" ❌ → "지난 1년에 몇 번" ✅.
- **행동>의향 워딩**: "쓰실래요?" 최소화, "~한 적 있어요?"로. 불가피한 의향 문항은 신호 등급을 낮춰 분석.
- **부정·탈출 보기**: "없다/안 함/모르겠다/다 무료여야"를 핵심 문항마다.
- **순서 랜덤화**: 복수선택·태도 보기. *단, 척도형·"기타"·"없음/단독선택"은 고정.*
- **loaded 워딩 제거**: "놓치면 큰일 나는" 같은 공포 프레이밍 금지(brand-voice 정합 + 응답 편향).
- **모바일 문장 길이**: 화면 2줄 내. 괄호 보조설명은 짧게.

## 4. WTP 심화 모듈 (선택 — *가격 책정 단계로 갈 때만* 켜기)

> ⚠️ 전문가 판단: **지금은 need 검증 단계라 main flow엔 행동 WTP(Q9)+가벼운 가격감각(Q12)만.** 아래 **Van Westendorp PSM**(가격민감도)은 *제품·가치를 응답자가 어느 정도 아는* 가격 책정 단계용. 콜드 서베이에서 미리 돌리면 허수 가격이 나옴. 베타 후/컨셉 충분 노출 후 별도 폼으로.

PSM 4문항(같은 응답자에게, 월 구독 가정):
1. **너무 비싸서** 살 생각이 안 드는 가격은? (too expensive)
2. **비싸지만** 그래도 고려는 해볼 가격은? (expensive)
3. **합리적이라** 느껴지는, 잘 샀다 싶은 가격은? (cheap / good value)
4. **너무 싸서** 품질이 의심되는 가격은? (too cheap)
→ 4 곡선 교차로 수용가격대(범위)·최적가격점 도출. (대안: Gabor-Granger = "X원이면 쓰겠어요?"를 가격 올려가며.)

## 5. 사전 분석 플랜 (데이터 보기 전 고정)

**문항 → 신호 매핑**
| 신호 | 1차(행동) | 2차(진술) |
|---|---|---|
| 진통제 실재 | Q3 빈도 · **Q4 복구불가** · Q9 과거지출 | Q1 온도 |
| 돌봄 pain(맞벌이) | **Q5 막막** | Q2 |

| job 무게(A vs B) | — | Q10 |

| PMF | **Q16 실제 이메일** | Q14 Sean Ellis · Q13 |
| referral 토양 | **Q16 남김** | Q15 |

**핫 ICP 정의(인터뷰 1순위)** = `Q1(조급/막막) AND Q3(놓침 有) AND Q16(연락처 남김)`. 여기에 `S3 첫째 + S2 맞벌이 + Q5 막막`이면 최핫.

**판정 임계(5~10명 인터뷰와 합산, n 작으면 방향 신호로만)**
- ✅ **진통제 강화**: Q4 '복구불가' + Q9 '과거 지출有'가 응답자의 **과반** / Q5 '막막' 맞벌이의 과반.
- ✅ **PMF 신호**: Q14 '매우' **40%↑** *그리고* Q16 이메일 전환 **20%↑**(진술-행동 동반일 때만 인정).
- ⚠️ **재조준**: Q10이 A/B 한쪽 쏠림 → 쏠린 job으로 좁힘. Q2·Q6에서 돌봄이 압도 → 돌봄 공백을 1급 쐐기로.
- ❌ **비타민 경고**: Q14는 높은데 Q9 과거지출·Q16 이메일이 바닥 → *말만 좋음*(politeness). 가설 약화로 판정.

**해석 가드**
- **진술-행동 괴리 우선순위**: 충돌 시 *행동(Q4·Q9·Q16) 채택*. Q13·Q14·Q15는 부풀려진다고 가정.
- **"좋아요/괜찮다"는 집계 제외**(§8.1 politeness). 카운트는 사실·돈·실제 행동.
- **표본 한계 명시**: n<100은 *유의성 아닌 방향*. 맞벌이×첫째 서브그룹은 더 큰 n 필요 — 부족하면 "관찰" 표기, 단정 금지.
- ATT 오답 응답은 전 분석에서 제외.

## 6. 필딩 & 운영

- **파일럿 먼저(필수)**: 지인 외 5명에게 돌려 ⓐ 이해 안 되는 문항 ⓑ 이탈 지점 ⓒ "기타" 폭주 보기(=보기 누락 신호) 점검 → 본 배포 전 수정.
- **채널·쿼터**(§ICP_INTERVIEW §0.5-C): 수도권 맘카페·오픈채팅 1~2곳 깊게. 맞벌이×첫째가 과소표집되면 그 세그 추가 모집.
- **어뷰징·중복**: 인센티브가 있으면 무관 응답 유입 → ATT 체크 + (가능하면)"1인 1응답 제한"·이메일 중복 제거. 보상은 **인터뷰 신청자(Q16)에게만** 지급(서베이 자체 보상은 허수 유발).
- **이탈 모니터링**: 구글폼 응답수를 섹션별로 못 보므로, 중간 이탈 의심되면 문항 수 더 줄이기. 진행률 표시 ON.
- **퍼널 연결**: Q16 연락처 → [[ICP_INTERVIEW]] 핫 ICP로 즉시 컨택(서베이는 *발굴*, 인터뷰가 *판정*).
- **윤리·프라이버시**: 무기명·아동 PII 최소(이름·생일 수집 금지, '2020년생' 여부만). 연락처는 인터뷰 목적 명시 동의(PRIVACY).
