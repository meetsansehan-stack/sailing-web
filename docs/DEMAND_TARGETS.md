# 수요 발굴 — 표적 키워드 리스트 (3중 필터 통과)

> niche(검색량·경쟁) + affinity(타겟 적합도 skew) 결합. "이길 수 있고 + 의도 뾰족 + 진짜 우리 유저(0~8 부모)"인 키워드.
> 근거: `DEMAND_NICHE.md`(검색량·경쟁) + `DEMAND_DEMO.md`(연령·성별 skew). 자동 아님 — 데이터 기반 판단 정리.

## 선정 원칙 (검증된 핵심)
1. **"아이·어린이" 명시어만 우리 수요.** 같은 묶음 내 검증에서 일관:
   - 서울아이랑갈만한곳(1.45)·어린이박물관(1.40) ✅ ≫ 일산가볼만한곳(0.81)·6월가볼만한곳(0.48)·데이트가볼만한곳(0.73, 대조군) ❌
   - 받아쓰기(1.72)·전래동화(1.54) ✅ ≫ 한글맞춤법(0.52, 성인 본인용) ❌
   - 처음학교로(1.30, 부모) ✅ ≫ 아이돌봄사(0.33)·베이비시터(0.45, 공급자·구직) ❌
2. **검색량 크다 ≠ 우리 유저다.** bare `{지역}가볼만한곳`(19k)은 트래픽 커도 일반 나들이객 → 표적 제외.
3. skew는 *묶음 내 상대값* → 방향성(✅/❌)은 신뢰, 절대치는 근사.

## ⚠️ 강등·제외 (트래픽 있으나 우리 유저 아님)
- **돌봄 *인력*어**(아이돌봄사 0.33·베이비시터 0.45·아이돌보미 0.51) = 구직·공급자 검색. 수요자(부모) 아님 → 콘텐츠 우선순위 ↓. (제도 안내는 `아이돌봄서비스`로, 매칭은 제휴 BM 영역)
- **bare 지역 나들이어**(가볼만한곳·놀이공원·물놀이장) = 일반인.
- **성인 범용어**(한글맞춤법 등).
