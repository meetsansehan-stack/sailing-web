## 0. 한 줄 정의 \[확정 2026-05-21]

**북극성(앱 본체):**
> **내 아이를 알아가고 앞을 그려가는 시간 — "내 아이 청사진".** 금융앱 〈내 자산/신용〉 탭처럼, 아이의 *현재 차트(스케줄·방향성)*와 *미래 여정(blueprint)*을 한눈에. 경쟁이 아니라 이해로, 광고·공포 없이.

**웹 MVP(지금, 검증 surface):**
> **내 아이·내 동네에 맞춘, 검증된 육아 정보 30초 데일리 브리핑.**
> = 맞춤(C) × 안티-맘카페 신뢰(B) × 시의성 한 섹션(A). → 청사진의 *시간축 연료*.

## 0.1 제품 아크 (비전 → 단계) \[확정 2026-05-21]

```
[지금]  웹(PWA) = 맞춤 데일리 브리핑   → 청사진의 "시간축 연료" + 수요/신뢰 검증
[유입]  진단 테스트 (가벼운 웹페이지)   → 바이럴 그로스해킹 + 청사진 개인화 씨앗
[본체]  앱 = 내 아이 청사진 지도        → 현재 차트 + 미래 여정(공간축). 검증 후.
        + 알림/위젯 = 청사진 마일스톤 도래 알림 ("왜 앱?"의 정답)
[V2]    공유·커뮤니티·학원 정보교환 / BM(유료·광고 제휴)
        └ BM 1호 가설: 사교육을 편집 콘텐츠에선 "통계·데이터 비주얼화"로만 다루고(중립 데이터 매체),
          학원 정보·매칭은 직접 안 함 → 해당 서비스 제휴 유료 광고 모델. 편집/광고 명확히 구분 표기.
```

- **두 축 직교**: 브리핑=시간축("오늘 세상 소식"), 청사진=공간축("내 아이 여정 어디쯤·다음은"). 둘 다 C로 묶임.
- **단계가 서로를 받침**: 브리핑으로 정보 수요·신뢰 브랜드를 먼저 검증 → 큰 청사진 베팅 리스크 ↓.
- **앱 메인 = "내 아이 청사진 지도"(타임라인)**. 부모-아이 진단은 메인 아님 → 가벼운 웹 진단 테스트로 분리(그로스).

## 0.2 톤 원칙: "둘 다 품되 선 긋기" \[확정 2026-05-21] (제품의 영혼)

- **품는다**: 사교육·입학 준비 등 *현실 정보*를 회피하지 않음 (부모가 진짜 궁금한 것).
- **선 긋기**: 프레이밍은 경쟁·불안·과시가 아니라 **차분한 이해·발달 중심**.
- 운영 규칙(초안): "또래 대비"는 순위/등수 아닌 *참고 분포*. "안 하면 큰일" 공포 카피 금지. 학원은 추천/랭킹 아닌 *선택지·사실 정보*. → CLAUDE.md 콘텐츠 정책에 흡수 예정.

## 1. 문제의식: 순수 큐레이션은 약하다 \[확정]

- 큐레이션 뉴스레터 포맷은 검증됐지만(뉴닉) **남의 것** — 부모는 콘텐츠가 없어서가 아니라 **주의(attention)가 없어서** 못 본다. 이미 키즈노트·맘카페·유튜브·단톡방에 시간을 다 쓴다.
- "둘러보기"는 nice-to-have → 매일 올 이유가 약하다. **매일 와야 하는 이유 = 안 오면 손해 보는 구조**여야 한다.
- 따라서 차별화는 "콘텐츠를 더/잘 준다"가 아니라 **"다른 데선 안 생기는 결과를 준다"**에서 나와야 한다.

## 3. 타겟 페인 포인트 \[확정]

1. **놓침의 손해 (가장 강함)** — 늘봄·돌봄 신청, 국공립 대기, 검진, 지원금, 인기 공연 예매. 마감 지나면 1년 손해. 흩어져 있어 놓침. (*FOMO가 아니라 실손해*)
2. **불신 피로** — 맘카페의 광고·협찬·카더라·공포 마케팅. 믿을 1차 신호에 목마름.
3. **시간 빈곤** — 5분도 귀함. 긴 글·영상 못 봄. 30초 핵심.
4. **불안 증폭** — 정보 소비 자체가 스트레스. *덜 불안하게* 만들어주는 매체가 없음.

## 4. 차별화 웻지 — 후보 3 + 데이터 검증 \[검토중]

세 후보는 **경쟁이 아니라 적층(stack)**:

| 레버 | 역할 | 단독 후킹력 | 리스크 |
|---|---|---|---|
| **A. 키데이트 레이더 + 푸시** | 재방문 유틸리티 | **높음** (안 오면 손해 + 푸시로 찾아감) | 날짜 콘텐츠 밀도·정확도, 푸시 옵트인 |
| **B. 안티-맘카페 신뢰** | 브랜드 레이어 | 낮음(콜드 방문자가 30초에 못 느낌) | 포지셔닝≠기능, 매일 올 이유 X |
| **C. 연령·지역 개인화** | 관련성 필터 | 중간 ("내 아이/내 동네" 와우) | 세그먼트별 콘텐츠 깊이, 온보딩 마찰 |

### 4.1 데이터 검증: A의 연료 (시드 61건, 9일치)

| 지표 | 값 | 해석 |
|---|---|---|
| 날짜 임박형(마감+이벤트) | 16/61 = **26%** (주 ~12건) | 레이더 연료 **충분** |
| 순수 마감(신청 손해) | 3건 (4%) | 희소하나 고가치 |
| 이벤트(시작일) | 13건 (21%) | 연료의 대부분 |

**발견 1**: ★A를 "마감 알림"으로 좁히면 연료가 얇음(4%). **"이번 주 키데이트 레이더(이벤트+마감)"로 넓히면 충분(26%)**. → 후킹 프레임 = *"이번 주 갈 만한 것 + 가끔 놓치면 손해"*.

**발견 2**: 날짜 임박형은 거의 **지역 종속**(서울 키움·광주 ACC·DDP·국립중앙박물관…). → **A는 C 없이는 반쪽**. 둘은 엮여 있음.

### 4.2 방향 전환 (2026-05-21 논의)

초기 잠정 권고는 "보이는 후킹=A(레이더)"였으나, **A 단독 후킹이 약하다는 판단**(이벤트 나열은 둘러보기에 가깝고, 푸시는 무겁고 옵트인 난이도 높음)에 따라 **C 또는 B를 메인 후킹으로 격상** 검토.

**재구성 (C 주축 가설)**:
> 메인 후킹 = **"내 아이·내 동네 맞춤 30초 데일리 브리핑"** (C)
> - 신뢰(B) = 브리핑의 *재료 원칙* (검증 출처·광고/공포 ZERO) → 브랜드 레이어
> - 시의성(A) = 브리핑 안의 *한 섹션* ("이번 주 우리 동네 챙길 거"), 푸시는 V2

**C·B 단독 메인의 공통 약점**: "왜 *오늘* 와야 하나"에 약함(A의 강점이었음) → 데일리 개인화 브리핑으로 보완.

**핵심 함정**:
- C → 콜드스타트(세그먼트별 빈 화면). 완화: 개인화를 "빼기 필터" 아닌 **"정렬·강조"**로. 전국 공통은 모두에게, 지역 의존만 내 동네 우선.
- B → 콜드 방문자가 30초에 신뢰 못 느낌. acquisition 약함(위생요인 가능성).

### 4.3 확정 \[2026-05-21]

- **메인 후킹 = C 주축 "내 아이·내 동네 맞춤 데일리 브리핑".** B = 브랜드 레이어, A = 브리핑 속 한 섹션.
- **시작 지리 = 수도권 먼저** (서울·경기·인천). C 주축 + 이벤트 수도권 집중 + 솔로 리소스. 검증 후 확장.
- **⚠️ 스코프 변화**: 개인화(연령·지역)가 **V2 → V1 핵심**으로 승격. 기존 MVP 메모(`개인화는 V2`)를 의도적으로 덮어씀.

### 4.4 제품 함의 (C 주축 → MVP 재형성)

- **온보딩**: 익명 기본 + 점진적 프로파일("동네·아이 나이 설정하면 더 정확"). 강요 X.
  - 연령: 0–2 / 3–5 / 6–8 (형제 다중 선택 허용 검토).
  - 지역: 수도권 시·군·구 단위.
- **홈 = 맞춤 데일리 브리핑**:
  - 전국 공통(정책·발달·도서)은 모두에게.
  - 지역 의존(이벤트·시설·지역정책)은 **내 동네 우선 정렬·강조** (빼기 필터 ❌).
  - "이번 주 우리 동네 챙길 거" 섹션 = A(키데이트)를 지역·연령 스코프로.
- **신뢰 UX (B 레이어)**: 출처 배지·credibilityScore 가시화, "광고 없음" 선언, 근거/원문 링크, 차분한 톤(공포 카피 금지).
- **개인화 메커니즘 = 정렬·강조** (하드 필터 아님) → 콜드스타트/빈 화면 방지.

## 8. 실데이터 진단 & 비타민→진통제 \[2026-05-27]

> 메일링 "아이의 항해"의 첫 실데이터로 §3·§4 가설을 두들겨본 결과. **결론: 병목은 TAM이 아니라 공명+유통이고, 지금 제품은 "비타민"으로 받아들여진다.** §3(놓침의 손해)·§4(키데이트 레이더)·§0.2(선 긋기)와 수렴 — 실측으로 강화됨.

### 8.1 실데이터와 신호 해석
- **데이터**: 2개월차, 주 1회, 구독 16명(대부분 지인), 오가닉 0, 오픈율 50%+. 구두 피드백 ~4명 = "내용 너무 좋다"·"유익하게 잘 본다"·"내용 넘 좋다"(긍정 3) + "좋은데 크게 안 와닿음"(미온 1).
- **해석 \[2026-05-29 교정]**: 구두 신호 5개 중 *어느 것도 PMF에 load-bearing 하지 않음* — 그리고 **편향은 긍정 쪽이 더 심함**. 지인의 "너무 좋다" = politeness default(데이터셋 최저 신뢰도). 따라서 매몰 위험은 부정 1개보다 *값싼 긍정 3개* 쪽이 더 큼.
  - "좋다/유익하다"는 **품질 만족 축**, PMF는 **필요(없으면 곤란) 축** — 둘은 직교. 훌륭한 콘텐츠(높은 "좋다")가 동시에 비타민(낮은 "없으면 곤란")일 수 있고 그게 지금 위험. "안 와닿음"이 진단적인 건 *부정이라서가 아니라 유일하게 need 축을 건드려서*.
  - 행동 신호도 결론 불가: 오가닉0 = 유통 활동 0의 당연한 결과(증거의 부재, "실험 미실행"), 오픈율 50% = engagement이지 need 아님.
  - **∴ 진단 상태 = "비타민으로 *판명*"이 아니라 "PMF 신호 미확보(어느 방향도 결론 못 냄) + 비타민 *위험 가설*".** n=16 지인 웅덩이에서 찻잎 점치기를 멈추고 깨끗한 측정(§8.4)을 떠와야 함.
- **그래도 진통제 쐐기에 베팅 유지**: PMF가 unknown이어도 진통제는 (a) 공명·(b) 수익화 확률이 둘 다 높아 비타민 경로를 EV로 dominate. §0(큐레이션 차별성 부족) 피벗 트리거와 수렴.
- **PMF 측정 정석(다음 실험에 적용)**: "좋아요?"가 아니라 **"내일 사라지면 얼마나 곤란?"**(Sean Ellis 테스트, "매우 실망" 40% 임계) + **"다른 부모에게 전달한 적 있나?"**(행동). 후자가 오가닉0의 진짜 검증.

### 8.2 시장 규모 (TAM→SAM→SOM) — 연령 0~8 유지 근거
- **TAM**: 전국 0~8세 아동 ~240~250만 / 가구 ~170만. 수도권 가구 **~85~90만**.
- **SAM(수도권 깔때기)**: 가구 90만 ×디지털활성 50% ×유료의향 5~10% = **유료 후보 ~2.3만~4.5만**.
- **SOM**: 보수 1,000가구(~₩70M/년) / 기본 3,000(~₩210M) / 낙관 8,000(~₩760M). 0.3%만 잡아도 솔로 흑자권.
- **결론**: **TAM은 병목이 아니다.** 매출을 가르는 건 유료 전환율×ARPU×BM이지 연령 폭이 아님.
- **연령 확장 판단 \[확정]**: 0~8 **유지**. "0세~초등 전체"는 싼 움직임(0~2, 같은 감정세그)+비싼 움직임(초등 고/입시, 브랜드 DNA 충돌·다른 고객·콘텐츠 희석)의 번들 → 후자는 PMF 전 금지. 저출생 환경 = 머릿수 확장은 인구 추세와 역행, **가구당 깊이·ARPU(청사진·개인화·제휴)가 정답**. 0~8은 LTV 창이 길어(최대 8년+) 단위경제 우수.

### 8.3 해결 방향 — job을 비타민→진통제로
- **축**: "놓친 *뉴스* 캐치업"(비타민) → "놓치면 큰일 나는 *신청·마감·입학 준비*"(진통제). 같은 surface·인프라, **조준만 변경**(재구축 아님).
- **진통제 쐐기 \[가설]** = **A(초등 입학 준비 여정) + B(신청·마감 레이더: 돌봄·늘봄·방과후·추첨) 결합**:
  > "취학 전후(5~8세) 내 아이 기준으로, 놓치면 안 될 신청·마감 + 입학 준비 여정을 손에 쥐어주기"
  - 놓치면 복구 불가(자리 끝) = 진짜 진통제 / 마감일 = 명확한 urgency / 청사진 와우 페르소나(5~8) 정조준 + 항해 메타포 + 죄책감 면죄부 + 선 긋기 = 완벽한 브랜드핏.
  - **이미 만든 부품(`eventStartDate`/`eventEndDate`/`deadline` 스키마·키데이트 캘린더 배너)이 이 job의 엔진.** = §4 키데이트 레이더(A)를 "취학 전후"로 좁힌 날카로운 버전.
  - 정부24·처음학교로·맘카페에 파편화 → "내 아이 일정으로 통합·개인화"한 곳 없음 = 빈틈. 청사진 "교육레일 레이어"의 손에 잡히는 MVP 한 조각.
- **회피/주의**: 발달 불안(D, 브랜드·의료 리스크 → "참고 분포" 톤만) / 사교육 시기(E, 선 긋기 → 통계+제휴BM으로 격리).

### 8.4 다음 검증 (코드·결제 0원)
1. **메일 리포맷 실험**: 취학 전후 세그에 "이번 주 놓치면 안 될 신청·마감 3 + 입학 준비 1" 형태 → 반응 델타(답장·포워드·능동 요구) 측정. "안 와닿음"이 "챙겨야겠다"로 바뀌나.
2. **지인 밖 ICP 인터뷰 5~10명**: 어느 진통제 job에 몸이 기우는지(입학·신청 불안) 검증.
- ⚠️ 이건 §6 "검증 1순위(파이프라인 날짜 밀도)"보다 **상위 게이트**: 큐레이션을 더 만들어도 비타민이면 PMF 안 옴. 결제·코드 없이 지금 가능 → "검증 우선" 철학에 부합.

## 9. 출시 순서 & Revenue 방향 \[확정 2026-05-29]

> CPO(사용자) 정리 + 스파링. 핵심 = "출시 전에 revenue를 *결정·구현*하는 게 아니라, 미래 옵션을 *닫지 않게 방어*한다."

### 9.1 출시 순서 \[확정]
- **웹(모웹 포함) MVP → PMF → 앱(청사진 = wow) → growth/viral.** 앱이 "왜 앱?"의 와우, 웹은 검증 surface(§0.1).
- **MVP 정의 교정**: "내가 고객으로서 최소한 필요한 것"이 아니라 **"진통제 쐐기 1개"**(취학 전후 신청·마감 레이더, §8.3). scratch-own-itch는 sub-PMF 상태에서 *편안한 캐치업(비타민)*으로 드리프트하기 쉬움 — 코어는 "있으면 좋은 것"이 아니라 "놓치면 손해"여야.
- **AARRR 컷**: Activation + Retention = **MVP 내장**(retention 루프 "왜 다시 오나" = PMF 테스트 자체, 메커니즘 = 시점 기반 relevance = 키데이트 캘린더가 엔진). Referral·Revenue = **그 뒤**. 공명 전 referral = 깨진 양동이에 물 붓기.
- growth 에이전트 추가 여부 = **출시 후** 임팩트 논의.

### 9.2 Revenue 방향 \[확정: 방향만, 구현·가격 No]
- **모델 = freemium, 단 free와 paid가 *서로 다른 job*** (free→paid 거부감의 원인 = "있던 걸 뺏기"; *증축*에는 거부감 없음):
  - **Free core (영원 무료)**: 캐치업 + 마감 레이더(진통제). acquisition·retention·신뢰 엔진. 유료화 금지.
  - **Paid (나중 *증축*)**: 앱 청사진 / 개인화 깊이 / 플래닝. **wow를 유료로.** 전환 아닌 증축이라 "공짜 주다 갑자기 유료" 시나리오 자체가 안 생김.
- **출시 전 결정 2개 (0원·코드 없음):**
  1. **"전부 영원 무료" 약속을 카피에서 하지 말 것** (무료 *core*는 약속, *전부*는 약속 안 함) = 미래 거부감 백신.
  2. **신뢰의 벽(편집 vs 제휴) 처음부터 설계** = 미래 BM 신뢰성의 선결조건 (§0.1 BM 1호·§0.2 선 긋기와 동일선).
- ⛔ **출시 전 금지**: 페이월·광고 슬롯 구현, 가격 책정. MVP = 무료·수익화 0. (growth 배경상 PMF 전 퍼널·수익화 과설계가 최대 유혹 — 병목은 funnel 아니라 공명.)
- **광고 모델 (AI 시대 질문에 대한 답)**: CPM/노출 광고 = **No** (신뢰 해자 직접 갉아먹음 + zero-click·AI가 노출 가치 잠식 중). intent/transaction = **Yes** (부모가 신청·예약을 *결정하는 순간*의 제휴·예약·검증된 lead-gen with 신뢰벽). = BM 1호(학원 제휴)를 "노출"이 아니라 **"의도·거래"**로 재정의. AI가 무너뜨리는 건 일반 콘텐츠/집계 가치, 못 무너뜨리는 건 신뢰 큐레이션×개인화 타이밍×행동.
- **★ 핵심 연결**: 진통제는 과금 가능, 비타민은 불가 → **"돈 낼까?"(revenue)와 "PMF인가?"는 쐐기에선 같은 질문.** ∴ 별도 수익화 워크스트림 불필요 → **§8.4 ICP 인터뷰에 WTP probe 1줄 접합**으로 PMF + 수익화 신호 동시 획득.

### 9.3 우선순위 주의
- 이 절은 *방향 합의*이지 *지금 할 일*이 아님. revenue는 검증에 probe로 올라타고, 결과가 나온 뒤 구체화.
- **★ 시퀀싱 교정 [2026-06-01]**: 기존 "검증이 출시보다 위 게이트"를 **"베타 출시-first → 검증을 *베타 직후*로"**로 조정. 근거 = MVP가 이미 대부분 지어져 출시 한계비용이 낮고(경고의 "몇 달 짓지 마라" 비용이 sunk), 라이브가 유통·행동·WTP를 더 잘 학습시킴(§8 병목=공명+유통은 라이브 아니면 못 배움). **베타 = 대놓고 잠정적 학습 프로브**라 "출시=PMF 착각" 함정이 소멸.
  - **성패 가드레일 3 (안 지키면 비타민 퍼널 최적화 함정)**: ① MVP에 *진통제*("미리 준비"/마감 레이더)가 1급으로 포함 — 순수 큐레이션만 띄우면 비타민을 검증함. ② **유통과 한 묶음**으로 베타 — 안 그러면 트래픽 낮음이 "비타민이라"인지 "유통 안 해서"인지 구분 불가(§8 오가닉0 재현). ③ **kill/learn 기준을 베타 전에 박기**(리텐션 곡선·"미리준비" 재방문/푸시 옵트인·Sean Ellis 40%) + "개선(퍼널)"이 "필요 검증(need/WTP)"을 덮지 않게 — need·WTP는 라이브여도 *물어봐야* 나옴(인터뷰·in-app micro-survey).
  - **§8.4 검증 = 베타 직후 4주**로 이동: 행동(분석 자동) + need/WTP(실유저 인터뷰 5~10 + WTP 3층 probe).

## 10. AI 방어선 & 데이터 해자 웻지 \[2026-06-01]

> "AI/제로클릭이 콘텐츠를 잠식하는데 이 사이트를 만들 필요가 있나?"를 정면으로 두들긴 결과. **결론: AI는 *비타민 절반(콘텐츠)*을 죽이고 *진통제 절반*을 검증한다 — 피벗을 가리키는 3번째 독립 신호.** §8(비타민→진통제)·§9(거래 BM)와 수렴.

### 10.1 AI가 먹는 것 vs 못 먹는 것
- **AI가 먹는다(= 해자 X, 무료 미끼로)**: 정보 종합·개인화 환산·단발 체크리스트 = 우리가 이미 비타민으로 진단한 **캐치업 콘텐츠**가 정확히 이 부분. 커모디티·제로클릭·AI 종합에 잠식.
- **구조적으로 못 먹는다(= 능력 아닌 *위치* 문제라 안 닫힘)**:
  1. **트리거 소유 (push ≠ pull)** — AI는 물어봐야 답함. 핵심 통증은 "마감 날을 *물어볼 줄 모름*". 진통제는 먼저 찾아오는 것 → 해자 = "알맞은 순간에 부모를 찔러도 되는 신뢰·권한"(관계 자산). ⚠️ 선제형 에이전트 AI가 노리는 자리 = 가장 실질적 위협, *타이머 있음* → 좁은 쐐기를 일반 AI가 신경 쓰기 전에 신뢰로 선점.
  2. **검증된 책임(accountability)** — "틀리면 자리 끝"에서 AI 환각은 치명, 그리고 AI 제공자는 책임을 *명시적으로 거부*("직접 확인하세요"). "안 놓쳤다"를 누군가 책임져야 = 모델 아닌 제품·브랜드. 구조적으로 회피되는 자리.
  3. **하이퍼로컬·비색인 데이터** — 구청별 추첨·학교별 돌봄·처음학교로 마감은 열린 웹에 깨끗이 없음(로그인 뒤·PDF·매년 변경). 발로 뛴 검증·정리 = AI 검색이 자동으로 못 갖는 데이터 해자(§4.1 "날짜 임박형=지역 종속"과 동일).
  4. **또래 행동 데이터** — §10.3. AI가 가장 못 주는 것.
- **휴리스틱(모든 기능에 적용)**: "부모가 ChatGPT에 한 번 물어 얻나? → Yes면 무료 미끼지 해자 아님. AI가 책임 안 짐 + 먼저 찾아가야 함 + 로컬 검증 필요 → 그게 제품."
- **평결**: "AI로 쓰는 콘텐츠 사이트"면 만들지 말 것. 트러스트·트리거·하이퍼로컬·또래데이터 4축에 해자를 걸 때만 빌드.

### 10.2 수익화 재정의 — 콘텐츠는 무료, 돈은 거래에서
- **무료 콘텐츠 = 성장 엔진** (파이프라인 한계비용 낮음 + SEO·신뢰). AI에 먹히니 *효율적으로*만, 과투자 X.
- **유료 콘텐츠 직접 제작 = 솔로에 과부하 → 안 함.** 프리미엄 콘텐츠(=비타민+production 부담+AI에 먹힘) 금지.
- **수익화 = intent/거래** (§9.2). 국내 레퍼런스 클래스가 정함: 자란다·째깍악어(매칭)·똑닥(예약)·커머스 = 거래. 정보·커뮤니티(맘카페)는 무료, 돈은 거래·광고가 법.
- **★ 개인화/벤치마크는 *유료 상품이 아니라* 거래 BM의 전환율을 폭발시키는 *무료 데이터 엔진*.** (벤치마크 자체는 결제가 약함 — §10.3) 돈을 직접 버는 게 아니라 돈 버는 거래를 가능케 하는 자산.

### 10.3 데이터 해자 웻지 — 익명 또래 분포 \[검토중, §4 C레버 진화형]
- **페인**: "내 아이 스케줄 설계, 내가 맞게 하나? 다른 부모는 어떻게?"(확인받고 싶은 심정) = **AI가 구조적으로 절대 못 주는 가치** (또래 *실제 행동* 데이터는 모으기 전엔 세상에 없음 = proprietary 네트워크 효과).
- **보너스**: 연속적 통증(학기·방학·매주) → 취학(계절성)이 못 풀던 **retention/구독 churn 문제를 해결.** 취학=유입(acquisition), 벤치마크=1년 내내 retention·데이터 축적. 둘은 합체.
- **설계 가드 3개 (안 지키면 맘카페 복제)**:
  1. **커뮤니티 ❌ → 익명 벤치마크 ✅** (#1 입력→#2 분포). 풀 UGC/Q&A는 콜드스타트+모더레이션=솔로 함정·V2. "나를 위해 기록"하면 분포는 부산물(Strava 모델) → 입력 트리거 해결, 공개게시 없음=모더레이션·과시 나선 없음.
  2. **순위·백분위 ❌ → 참고 분포 ✅** ("이 시기 보통 ~함, 정답 없음"). 불안 낮추는 도구 vs 올리는 도구를 가르는 한 끗(CLAUDE.md "또래 대비=참고 분포"·죄책감 면죄부).
  3. **학원 이름·금액 ❌ → 시간·활동유형 ✅**. 학원은 선 긋기 라인 + 광고 BM 레이어.
- **WTP 진실**: 벤치마크 *숫자* 자체는 결제 약함(정보=커모디티+일회성 조회+불안 환기). WTP는 *해소*(계획·안심·프레이밍)와 *거래*에 붙음. **벤치마크=무료 후크/데이터 자산, 결제=그 위의 해소 구독 또는 거래.** 베팅 순위 = 거래 > 해소 > 정보(무료), 단 §8.4 WTP probe로 검증.
- **데이터 아키텍처**: 식별 개인기록=로컬(서버0) / 익명 집계(시군구+연령밴드+활동유형 시간)=서버=해자. 클라-only로는 분포 못 만듦. 상세 메모리 `mvp-account-data-architecture`.

### 10.4 다음 검증 (§8.4에 접합)
- ICP 인터뷰 WTP probe 3층 동시 측정: ⓐ 또래 분포 보는 것에 월 X원(→아마 No, 확인) / ⓑ 분포+내 아이 시즌 계획·리밸런싱이면(→해소 WTP) / ⓒ 거기서 발견한 체험·돌봄 예약·제휴면(→거래 WTP). 돈이 정보·해소·거래 중 어디 붙는지 데이터로.

## 11. 체류시간 지표·원문 링크·아카이브 보존 \[2026-06-02]

### 11.1 체류시간 = 유효 지표 (몰입 vs 가둠 구분)
- 읽기 깊이·재방문 = 충성도·전환 propensity → intent·transaction BM(§9·§10.2)과 정합(몰입 유저일수록 향후 유료·제휴 전환↑). **체류시간을 유효 지표로 채택.**
- 단 **몰입 체류(가치 있어 더 읽음=좋음) ≠ 가둠 체류(나갈 문 막아 머묾=허수·신뢰↓).**
- **레버 = 콘텐츠 질·관련기사·시리즈·내부 회유(recirculation)**, *아웃바운드 차단 아님.* → 로드맵 항목: 다음 읽을거리·관련 기사·토스식 섹션 다양화(`docs/toss-feed-analysis.md`).

### 11.2 원문 링크 = 유지 \[확정]
- 신뢰("중립 데이터 매체"·투명성) + 법적(어트리뷰션) + 정책(CLAUDE.md "모든 기사에 원문 링크 필수") + SEO(E-E-A-T) — 모든 축이 유지. 제거는 자해.
- 새 탭(`target=_blank`)이라 체류 손실 거의 없음(이탈 아닌 병렬). 클릭=요약 읽고 검증하는 만족 신호.
- **링크 로트 대응**: `factcheck.ts` 확장한 링크 헬스체크 + graceful degradation(죽으면 버튼 제거+안내). 자체 body가 durable artifact라 원문 죽어도 콘텐츠 생존(soft-404는 best-effort).

### 11.3 아카이브 보존 기간 \[기획 확정 2026-06-02]

**리서치 근거**: 뉴스 half-life 8h(일반)·20h(심층), shelf-life ~1.7~2.7일(Al Jazeera) — 캐치업 가치는 며칠 내 소멸(우리 시의성 윈도우 1주와 정합). 단 콘텐츠타입별 수명 상이(에버그린 가이드=수년 / 이벤트=행사일에 사망). SEO 업계 표준 = content pruning(오래된 저가치는 삭제 아닌 `noindex`, 3~6개월 주기 점검).

**핵심 = "보존"은 3개의 다른 결정이다 (분리하라):**
| 결정 | 비용 | 정책 |
|---|---|---|
| ① 저장(DB) | **비용 0** (요약만 저장: 1건 ~1.6KB, 10년 ~250MB ≪ Supabase 무료 500MB; 이미지는 URL만; 무기한=코드 0) + **데이터 해자 원천(§10.3)** | **무기한 보관** (삭제 No) |
| ② 노출(아카이브 UI) | 유저 관련성·브랜드 | **콘텐츠타입별 은퇴** |
| ③ 색인(SEO) | 색인 비대 vs 롱테일 | **에버그린 색인 / 만료 noindex** |

→ 직관 교정: "너무 많이 저장 말자"는 빗나간 조준. **저장은 싸다.** 진짜 레버 = 노출·색인 라이프사이클.

**콘텐츠타입별 은퇴 규칙:**
| 타입 | 수명 | 처리 |
|---|---|---|
| Event | 행사일 사망 + 링크로트 자석 | `eventEndDate`+버퍼 지나면 **노출·색인 제외**(최단) |
| Policy | 대체 전까지(수개월+) | 유지, "대체됨" 점검 |
| Market/Insight | 중간(~수개월) | 점진 noindex |
| Guide | near-에버그린(입학준비=매년) | **유지·색인**(롱테일 SEO 자산) |

**원칙**: 홈/캐치업=최근만(시의성 1주, 이미 처리). 아카이브 역할=캐치업 아닌 **레퍼런스·브라우즈·SEO**로 재정의. 죽은 링크는 §11.2 graceful degradation(낡은 아카이브가 깨진 링크 안 보이게 = 브랜드 보호). 구현 변수는 기존 필드(`contentType`·`eventEndDate`·`publishedAt`)로 대부분 계산 가능. 점검 주기 3~6개월. 상세 동작 = SPEC §9.
