## 1. 사이트맵 (계층)

```
Sailing Web
├─ 1차 (소비자 핵심)
│  ├─ /               세일링 뉴스 (홈·캐치업)  ★ 주 진입
│  ├─ /radar          미리 준비 (입학 신청·마감 레이더)  ★ 진통제 쐐기
│  └─ /reservations   예약 정보 (장소: 유형·가격·연령·지역 필터)
├─ 2차 (탐색)
│  ├─ /articles       모든 기사 = 통합 아카이브 (필터·검색)
│  ├─ /articles/[id]  기사 상세 (카드뉴스 본문)
│  └─ /issues/[date]  이슈 퍼머링크 (부가 — 1차 동선 아님)
├─ 계정 (소셜·선택·progressive)
│  ├─ /login          로그인 (카카오·구글)
│  └─ (구독·저장·싱크 진입점은 본문 내 CTA)
├─ 보조
│  ├─ /about          소개
│  └─ /letter         세일링 레터 (이메일 V2 — '준비중' 노출)
├─ 상시 배너 (GNB 하단 전폭)
│  └─ WeeklyHero      이번 주 키데이트 (이벤트·진행중 캘린더)
└─ 운영자 (분리)
   └─ /admin          대시보드·발행 이력 등 (소비자 IA와 격리)
```

### 결정 로그 (2026-05-29)
- **운영자 분리**: 로그인·대시보드 등 운영자 surface는 `/admin`으로 격리. 소비자 IA에 노출 X.
- **WeeklyHero ↔ 미리 준비 = 역할 분리 유지**: 시간축·잡이 다름(아래 §3). 시각적 중복만 축소.
- **아카이브 = 기사 중심 통합**: `/articles`가 정식 아카이브. `/issues/[date]`는 퍼머링크로만(1차 동선에서 내림). 홈 하단 동선 = "모든 기사 보기" 일원화.
- **스텁 노출 유지**: 소개·세일링 레터를 '준비중'으로 노출(세일링 레터는 이메일 의향 수집 surface).
- **부모 계정 = 소셜 로그인(카카오·구글) · 선택 · progressive** (CLAUDE.md 데이터·프라이버시 2026-05-29 수정). 계정 ≠ 아동 PII — 서버엔 부모 신원만, 아동 PII·연령/지역 개인화는 로컬-퍼스트 유지.

## 2. 글로벌 내비게이션 (GNB)

- **좌측**: ⛵ "세일링이 궁금해요"(/about) · 세일링 레터(/letter)
- **중앙**: Sailing 워드마크 → /
- **우측**: 최신 뉴스(/) · 미리 준비(/radar) · 예약 정보(/reservations) · 로그인(/login)
- **GNB 하단 전폭 배너**: WeeklyHero(이번 주 키데이트, collapse/expand) — 전 페이지 공통
- **푸터**: 서비스(홈·모든 기사·예약·세일링 레터) / 정보(소개·개인정보처리방침·문의)

> 로그인은 **벽이 아님** — 익명으로 모든 읽기 가능, 구독·저장·싱크 시점에만 로그인 유도(progressive).

## 3. 두 "시점" surface의 역할 구분 (중복 해소)

| | WeeklyHero (상시 배너) | 미리 준비 (/radar) |
|---|---|---|
| 잡 | 둘러보기 — "이번 주 갈 만한 것" | 진통제 — "내 아이가 놓치면 안 될 신청·마감" |
| 시간축 | 이번 주(단기, 이벤트 진행/마감) | 입학 여정(중장기, 시즌 마감) |
| 대상 | 전체 공통 | 내 아이(유치원/초등 세그) |
| 성격 | 캘린더·발견 | 체크리스트·행동 |

→ 둘은 다른 잡이라 공존. 단 같은 화면에서 시각적으로 헷갈리지 않게(색·위치·라벨) 분리.

## 4. 핵심 UX 플로우

### ① 캐치업 (주 잡, 5~10분)
```
진입(홈) → "오늘의 큐레이션" 스캔 → 관심 카드 → 기사 상세(카드뉴스) → "다 봤어요" 완결감
                                            └→ (더) 이전 기사 / 모든 기사
```

### ② 미리 준비 (진통제 쐐기)
```
홈 hero 배너(🧭) 또는 GNB '미리 준비'
  → /radar → 유치원/초등 토글 → 시기별 신청·마감 카드 펼침 → 출처(1차) 확인
  → (로그인 시) 내 아이 세그 저장·알림 [progressive]
```

### ③ 둘러보기 / 탐색
```
홈 카테고리 탭 · 이전 기사 → 모든 기사(/articles, 필터·검색)
예약 정보(/reservations) → 유형·가격·연령·지역 필터 → 장소 상세
```

### ④ 구독·계정 (progressive, 선택)
```
익명 읽기 → 구독/저장/싱크 욕구 발생 시점 → 카카오·구글 로그인
  → 서버: 부모 신원(이메일·소셜ID·동의)만 / 로컬: 아동 개인화 입력 유지
  → (V2) 이메일 브리핑·푸시
```
