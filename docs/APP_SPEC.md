## 0. 한눈에 (전략에서 락인된 것) \[확정 2026-05-21]

- **앱 = 본체. 북극성 = "내 아이 청사진".** 금융앱 〈내 자산/신용〉 탭처럼 아이의 **현재 차트(스케줄·방향성) + 미래 여정(blueprint)**을 한눈에 시각화.
- **앱 메인 기능 = 청사진 타임라인** + **마일스톤 도래 알림/위젯**("왜 앱?"의 정답).
- **웹(PWA, 지금 빌드 중) = 맞춤 데일리 브리핑** = 청사진의 *시간축 연료* + 수요/신뢰 검증 surface. 앱 안에선 *한 탭*으로 흡수.
- **부모-아이 진단(MBTI/육각형)** = 메인 아님 → *가벼운 웹 진단 테스트*로 분리 = 바이럴 그로스해킹 + 청사진 개인화 씨앗.
- **톤 = "둘 다 품되 선 긋기"**: 사교육·입학 등 현실 정보는 품되, 프레이밍은 *차분한 이해·발달 중심*(경쟁·불안·과시 배제). "또래 대비"=순위 아닌 참고 분포.
- **페르소나 = 부모만(V1). 지역 = 수도권 먼저. 앱은 웹 검증 후 착수.**
- **V2**: 공유·커뮤니티·학원 정보교환 / BM(유료·광고 제휴).

## 0.1 두 축 직교 (왜 둘 다 필요한가)

- **브리핑 = 시간축** ("오늘 세상에 무슨 일") — 웹 MVP.
- **청사진 = 공간축** ("내 아이 여정 어디쯤·다음은 뭐") — 앱 본체.
- 둘 다 C(연령·지역 개인화)로 묶임. 브리핑이 청사진에 *시점마다 붙는 연료*.

## 1.0 질문1 결론 — 청사진에 무엇이 보이나 \[확정 2026-05-21]

청사진 = 아래 4종 레이어 + 현재차트 + 줌 타임라인. **포지셔닝 = 의료·발달 "추적기"가 아니라 교육·여정 "플래너".** (검진·접종·발달 이정표는 정부앱·육아수첩 레드오션이라 **지원 범주에서 제외** — 2026-05-21.)

**보이는 마일스톤 (4 레이어)**
- **① 교육 레일** — 유치원·초등 신청/입학, 학기·방학, 늘봄·돌봄·방과후, 교육정책 적용 시점(누리·늘봄 등). 생일+지역으로 시점 자동계산. **MVP 척추**(정적·정확·안전).
- **② 체험 플래너** — 주말·일상 체험(박물관·문화센터·전시·공연·키즈카페 등)을 **동네·연령 맞춤 추천 → 예약 → 내 아이 일정으로** 박음. STRATEGY ★3 "이번 주 우리 동네 챙길 거"+키데이트 레이더의 앱 진화형(둘러보기 → 개인 캘린더). 동적(웹 Event 파이프라인 재사용 + venue/예약).
- **③ 학습준비 선택시점** — 한글·수 노출 시기, 초등 적응(기본생활·자기조절), 늘봄/방과후 등 공적 선택지. **"선 긋기" 좁은 형태만**: 사교육은 **통계·데이터 비주얼화**(참여율·트렌드·분포)로만, 개별 학원 나열·추천 X, "안 하면 뒤처짐" 카피 X. 학원 정보·매칭은 직접 안 함 → 제휴 유료광고(BM, V2). 취학 타겟의 핵심 차별화. (→ CLAUDE.md 선 긋기 가드레일과 동일.)
- **④ 시의성 브리핑** — 정책·뉴스·트렌드 캐치업(읽을거리)이 타임라인 "지금" 지점에 붙음. 웹 큐레이션 재사용(새 파이프라인 X). ②가 "예약·행동"이라면 ④는 "읽기·캐치업".

**현재 차트** = "단계 + 준비도 요약" — "만6세3개월 / 이번 분기 챙길 것 N개 / 다음 큰 전환 = 초등 입학 D-180". (육각형·일상원형스케줄 후보는 기각.)

**시간 스케일** = 줌 가능 전체지도(~만8세) + 임박 하이라이트. 기본 뷰 = "다음 전환까지 D-day + 이번 주말 체험 + 이번 분기 준비도", 줌아웃 시 전체 여정.

**와우 페르소나 = 취학전~초등 저학년(만 5~8세).** "준비도 차트"·"선 긋기 톤"·교육 레일이 정점에 닿는 구간. ⚠️ 웹 브리핑 타겟은 0~8 그대로 유지(앱 페르소나 좁힘은 앱 한정 — CLAUDE.md "0~8/페르소나 분리 없음"은 웹 기준).

## 1.0.2 질문2 결론 — 데이터 출처·저장 \[확정 2026-05-21]

- **정적 레퍼런스 = 새 `packages/blueprint-data`** (타입 안전 TS 상수 + Zod). ① 교육 레일(학사·입학·정책 연례 일정) + ③ 학습준비 시점. 웹·앱 공유. DB·외부API 아님(정적·소량·룰 성격). 단 ③ 사교육 통계(통계청 등)는 같은 패키지에 두되 **빌드타임 갱신 스크립트**로 끌어와 커밋(실시간 조회 X).
- **동적 = 기존 웹 파이프라인 재사용**: ② 체험 플래너(Event 콘텐츠 + venue/예약) · ④ 시의성 브리핑. 새 콘텐츠 에이전트 X.
- **개인화 시점변환 = 로컬 우선 + 클라이언트 순수함수.** 아이 생일·지역은 기기 로컬(웹=IndexedDB, 앱=SecureStore), **서버에 아동 PII 0**(CLAUDE.md 금지사항·PRIVACY와 동일). `(생일,지역,정적룰)→날짜` 순수함수, 알림=로컬 스케줄. 맞춤 정렬 필요 시 식별정보 아닌 **거친 세그먼트(시군구+연령밴드)만** 전송.
- ✅ 저장·세그먼트·아동PII 원칙은 **CLAUDE.md "데이터·프라이버시 원칙(웹·앱 공통)"으로 이미 승격**(웹 PWA 온보딩 공통 적용). 상세는 docs/PRIVACY.md.

## 1. 이 세션에서 풀 열린 질문 \[열린질문]

청사진이 아직 추상적. 새 세션의 작업 = 아래를 구체화 → 개념 → 화면 → PRD.

1. ~~**청사진 타임라인에 *실제로 무엇*이 보이나?**~~ → **§1.0에서 확정** (4레이어: 교육레일·체험플래너·학습준비·시의성. 검진·접종·발달 제외).
2. ~~**데이터 출처 — 누가/어떻게 채우나?**~~ → **§1.0.2에서 확정** (`packages/blueprint-data` 정적 + 웹 파이프라인 동적 + 로컬 시점변환).
3. **온보딩** — 익명 기본 + 점진적 프로파일. 진단 테스트를 어디까지 결합?
4. **알림/위젯** — 마일스톤 D-day 푸시 + 홈위젯 스펙.
5. **기술** — `apps/mobile`(Expo/RN?) 모노레포 편입, `packages/shared` 타입 재사용. (착수 시점에)
6. **"선 긋기" 톤의 구체 규칙** — 사교육·또래비교를 차분하게 다루는 UI/카피 가드레일.
7. **아동 데이터 프라이버시·보안** \[중요] — 청사진은 아이 생일·발달·지역 등 *아동 민감정보* 수집. 개인정보보호법(만 14세 미만 법정대리인 동의 등) 의무. 앱 설계 초기에 `docs/PRIVACY.md`(아동 데이터 원칙) 씨앗 → 출시 직전 보안 게이트(/security-review)가 집행. 수집 최소화·로컬 우선 저장 등 원칙 선결.

## 1.2 앱 개발 전문 역할(에이전트) — 도입 방식 \[확정 2026-05-21]

상시 팀 X, **"필요할 때 부르는 on-demand 전문 패스"** (5/20 합의, [[agent-team-triggers]]). 같은 repo·컨텍스트 공유.
- UX 디자이너 패스 = 청사진 화면 설계 시(가장 임박). iOS/Android 개발 = 앱 빌드 착수 시(프레임워크 확정 후, 그 전엔 정의 무의미). QA·보안 = 출시 직전 게이트(빌트인 /verify·/review·/security-review·/ultrareview 우선). PM-보조 = 트랙 동시진행/결정 병목/출시 후.
- 커스텀 서브에이전트는 *반복 가드레일을 담을 때만* 1~2개(UX-review=브랜드+접근성, app-security=아동프라이버시 체크리스트). `.claude/agents/`(repo 공유)에 둠.

## 1.1 지침·에이전트 구조 (공통 vs 표면별) \[확정 2026-05-21]

- **공통 지침은 자동 상속**: `CLAUDE.md`·`MEMORY.md`는 이 repo의 모든 세션에 자동 로딩. 새 앱 세션도 그대로 물려받음. 추가로 `docs/STRATEGY.md`만 읽으면 전략 맥락 완비.
- **공통 정책 1개**: 브랜드·톤·콘텐츠·출처·데이터모델·코딩규칙 = `CLAUDE.md` 단일 소스. 포크 금지. (앱 착수 시 CLAUDE.md 상단에 "공통/표면별" 한 줄 정리만 권장.)
- **콘텐츠 에이전트 공유**: `agents/`(research/curation/writer/editor/hooking) + `packages/agents-core`는 웹·앱 공유. 앱 전용 콘텐츠 파이프라인 만들지 않음. 청사진 마일스톤은 *정적 레퍼런스 데이터*(새 `packages/` 또는 seed)로 — 새 에이전트 아님. 진짜 새 콘텐츠 잡 생기면 같은 `agents/`에 1개 추가(공유 core).
- **개발 에이전트 분리 불필요**: 같은 Claude Code가 모노레포 전체 커버. "분리"는 *세션(창)* 레벨(컨텍스트 위생)이지 에이전트 정의 레벨 아님.

## 2. 설계 산출물 목표

- 청사진 개념 정의 → 핵심 화면(타임라인/차트) 목업 → 데이터 모델(마일스톤 레퍼런스) → 앱 MVP PRD → `apps/mobile` 스캐폴드(검증 후).

## 3. 하지 말 것 (스코프 가드)

- 콘텐츠 파이프라인 새로 만들지 않기 (웹과 공유).
- 별도 repo로 쪼개지 않기 (모노레포 `apps/mobile`).
- CLAUDE.md 포크하지 않기 (제품 공통 정책 1개 유지).
- 공유·커뮤니티·BM은 V2 — MVP 스코프에서 제외.
