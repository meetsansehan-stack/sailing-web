# 링크 제휴(Affiliate) 비즈니스 모델 — 전략 리서치

> **상태**: 리서치·전략 분석 (2026-06-05). 도서 컬렉션의 커머스 레이어 사전 검토.
> **결론 요약**: 책 링크 제휴는 *지금의 수익원*이 아니라 *나중의 거래 레이어를 위한 배선*. 지금은 신뢰를 안 깨면서 길만 깔아둔다.
> **연결**: 도서 컬렉션 기획 · STRATEGY §10.2(수익=intent·거래) · 톤 가드레일(제휴 광고 격리).

---

## 0. 핵심 결론 (TL;DR)

- "링크 걸고 구매 수수료" = **CPS(성과형 제휴)** 모델.
- **책 제휴는 수익 기둥이 못 됨** — 수수료 3~5%, 저마진. ₩15,000 책 × 3% = 건당 ₩450.
- 의미 있는 수익은 ① 볼륨 ② **쿠팡 whole-cart**(세션 장바구니 전체) ③ 인접 고마진(육아용품)에서 나옴.
- **알라딘** = 표지 API + 셀프 3% 제휴를 한 번에 푸는 최소 저항 경로 (그래서 처음 제안).
- **교보·예스24**는 셀프 제휴 약함 → 수익화하려면 **링크프라이스 같은 네트워크** 경유.
- **네이버식 가격비교**는 표시는 가능하나 *수익은 별개 레이어*(각 스토어 제휴 필요), 지금 빌드 비추천.
- 제품 성격상 1번 리스크 = **이해상충**(편집 신뢰 vs 제휴 인센티브) → 방화벽 필수.
- **공정위 추천·보증 심사지침** = 제휴 표기는 *법적 의무*(모호 표현 위법, 위반 시 과징금·형사).

---

## 1. 모델의 정체 — CPS(성과형 제휴)

성과형 광고 과금 기준:
- **CPS** (Cost Per Sale) = 구매 시 수수료 ← 우리가 하려는 것
- CPC(클릭당) / CPA(가입·행동당) / CPM(노출당)

CPS에 연결하는 두 갈래:

| 방식 | 정의 | 예 | 특징 |
|---|---|---|---|
| A. 직접 제휴 | 그 회사와 1:1 | 알라딘 TTB, 쿠팡 파트너스 | 셀프 가입 쉬움, 단일 스토어 |
| B. 제휴 네트워크(어그리게이터) | 한 계정으로 여러 스토어 | 링크프라이스(국내 1위, CPS, 34만 제휴사, 도서 포함) | 여러 구매처 한 번에, 정산 통합 |

---

## 2. 구매처별 비교

### 알라딘 TTB
- **표지·저자·출판사·ISBN API** + **책 판매가 3% 수수료**(클릭 후 구매).
- 법인 회원 = 별도 계약으로 다른 수익율 가능, 사이트 파트너 추가 수수료.
- → "표지+수익"을 셀프서비스로 동시에 푸는 최소 저항 경로.

### 교보문고 · 예스24
- 셀프 제휴 프로그램이 알라딘만큼 정비 안 됨.
- ISBN 딥링크는 가능하나 **단순 연결(수수료 0)**.
- 수익화하려면 **링크프라이스 네트워크 경유**.
- 브랜드 적합성(서점=신뢰)은 매력.

### 쿠팡 파트너스
- 기본 3% (카테고리별 상이, 책은 낮은 편).
- **★ 라스트클릭 + 24h + 그 세션 장바구니 전체에 적용** (책 링크로 들어가 기저귀 사도 수수료).
- 가입 후 **누적 전환 15만원** 넘어야 API 활성(그 전엔 수동 링크).
- 종합몰이라 브랜드(중립·신뢰) 톤 관리 필요.

### 네이버 쇼핑 (가격비교 매핑)
- 네이버 쇼핑 검색 API로 **여러 구매처+가격 표시는 가능**.
- 단 그 비교 자체는 수익 아님 — 각 구매처 클릭이 돈 되려면 **그 스토어마다 제휴(네트워크)** 필요.
- 네이버 자체 모델은 셀러가 네이버에 내는 CPC → 우리가 네이버 데이터만 쓰면 수수료는 네이버가 먹음.

---

## 3. 냉정한 경제학 — 책 제휴는 수익 기둥이 못 됨

- 책 수수료율 3~5% (저마진). 알라딘 3%, 아마존도 책 3~4.5% + 최근 최대 50% 삭감(APAC 2025 말~).
- ₩15,000 책 × 3% = 건당 ₩450. 월 100권 = ₩45,000.
- → 책 링크 제휴 *단독*은 의미 있는 매출 X. 의미 있으려면:
  - (a) 볼륨(대형 트래픽) — 아직 없음
  - (b) 쿠팡 whole-cart — 거래 레이어(§10.2)에 가장 근접
  - (c) 인접 고마진(육아용품) — 책은 미끼, 수익은 옆 카테고리

---

## 4. 제품 성격 관점 — 진짜 고민할 4가지

1. **★ 이해상충 (1번 리스크)** — "중립 신뢰" 포지션 vs 제휴 인센티브(잘 팔리는/수수료 높은 책 추천 유혹). **편집 선정 ↔ 제휴 수익 방화벽 분리** 필수. 큐레이션 제품이 affiliate 붙일 때 항상 죽는 지점.
2. **신뢰 자산 > 한계 수수료** — 건당 ₩450 vs 브랜드 가치. 도서관 우선(공익) CTA가 신뢰 지킴.
3. **법적 의무 (공정위, 선택 아님)** — 추천·보증 심사지침: 경제적 이해관계를 제목/첫 부분에 명확히(본문보다 크게/다른 색, 잘리지 않게). "수수료를 받을 수 있습니다" 같은 모호 표현 위법, "수수료를 받습니다"처럼 명확히. 위반 시 시정·과징금·형사. → 우리 "명확 구분 표기"는 법적 요구였음.
4. **데이터 가치 > 직접 수수료** — 부모의 구매 의도 클릭 데이터가 ₩450보다 전략적으로 더 값짐(거래 해자·벤치마크 §10.3). 수익을 "지금 ₩" 아닌 "의도 데이터 축적"으로 프레임.

## 5. 전략 권장 (우리 케이스)

| 구분 | 권장 |
|---|---|
| 지금 | 카탈로그=신뢰·사용 검증용. 도서관 우선 + 제휴는 라벨 단 보조 링크. 과투자 X(트래픽 없으면 수익 0) |
| 시작 픽 | 알라딘(표지+셀프 3%) → 트래픽 생기면 쿠팡 파트너스(whole-cart) |
| 확장 시 | 멀티스토어·가격비교 원하면 링크프라이스(네트워크 한 계정으로 여러 서점) |
| 보류 | 네이버식 가격비교는 지금 X(빌드 큼·신뢰 한계이득·수익은 네트워크 필요) |
| 불변 원칙 | 편집·제휴 방화벽 + 공정위 표기 + 도서관 1순위 |

**한 줄 결론**: 링크 제휴는 *지금의 수익원*이 아니라 *나중의 거래 레이어를 위한 배선*. 지금은 돈 벌려고가 아니라, 신뢰를 안 깨면서 "거래로 가는 길"을 미리 깔아두는 관점. 직접 수익은 트래픽·쿠팡 whole-cart·인접 고마진에서.
