# 수요 발굴 (Demand Mining) — 부모 검색 수요로 큐레이션·유통을 정렬

> **목적**: 부모가 *실제로 검색·고민하는 것*을 데이터로 캐서 ① 큐레이션·발행 토픽 우선순위(**공명**) ② SEO 랜딩·시즌 캘린더(**유통**)에 반영. CSO 진단(비타민→진통제, sub-PMF)의 *공명+유통 병목*을 직접 겨냥.
> **척추 = 시즌성**. 진통제 쐐기(취학 전후 신청·마감)가 본질적으로 시즌성이고, 이미 만든 [웹 키데이트 캘린더 배너]가 실데이터 0건으로 비어 있음 → 수요 발굴이 그 배너에 실탄을 채움(신규 삽질 아님, 기존 자산 활성화).
> **원칙 정합**: 네이버 공식 API 2개만 사용(무료·약관 클린·PII 0) → 서버 최소·솔로 운영. 맘카페·자동완성 *대량 스크래핑은 비채택*(약관 회색+유지보수 부담).

## 엔진 = 네이버 공식 API 2개 (상보)

| API | 주는 것 | 못 주는 것 | 역할 |
|-----|--------|-----------|------|
| 검색광고 **키워드도구** (`/keywordstool`) | 연관키워드 + **절대 월검색량**(PC/모바일) + 경쟁도·CPC | 누가 검색하는지 | 발굴 + 규모 |
| 데이터랩 **검색어 트렌드** | **연령·성별·기기·시계열**(지수) | 절대량 | 시즌성·페르소나 검증 |

→ **반드시 조인**: "30대 여성이 이 시즌 급등시키는 키워드(데이터랩) × 그 절대 규모(키워드도구)" = 핵심 부모 페르소나의 진짜 수요.

## 시즌 캘린더 — 확정판 (2026-06-08 데이터 검증)

🔴 = 진통제 윈도우(신청·마감·예매 = 시점성 강함, 발행은 N주 *선행* 필요). ✏️ = 가설서 수정된 칸.

| 월 | 핵심 수요(검증) | 비고 |
|----|----------------|------|
| **1월** ✏️ | 🔴늘봄·초등돌봄·아이돌봄서비스(11.5만) 신청 정보 / 🔴초등 입학준비 / 한글·학습지 | **진통제 실측 피크**(신청 검색이 마감 선행) |
| **2월** ✏️ | 🔴늘봄(2월)·🔴유아학비 지원 / 🔴영어유치원 / 새학기 준비물 | 신청 마감기 |
| **3월** ✏️ | 새학기 봉: 분리불안·훈육·형제다툼·사교육비·전집·초등영어 | **정서·학습 동시 폭발(가설보다 광범위)** |
| **4월** | 봄 체험·나들이 / 어린이날 공연 예매 | |
| **5월** ✏️ | 어린이날·가정의달 나들이 | 공연 검색 피크는 5월 아닌 7월(아래) |
| **6월** | 여름 대비 / 물놀이 준비 | |
| **7~8월** ✏️ | 🔴여름 놀이·체험·공연 트래픽 **연중 최대**(워터파크 16.5만·키즈카페 18.6만·어린이뮤지컬·풀빌라·물놀이장·실내놀이터) | **베타 유입 엔진** |
| **9월** ✏️ | 🔴처음학교로(유치원 입학) 상승 시작 / 2학기 적응 / 가을 체험 | 입학 D-day 리드 시작 |
| **10월** ✏️ | 🔴도서관 프로그램 신청(강도9.4) / 가을 나들이·축제 | |
| **11월** ✏️ | 🔴처음학교로 **유치원 입학 신청 피크(~29만)** / 🔴어린이집 입소대기 / 🔴겨울 공연 예매 | 연중 최대 진통제 |
| **12월** | 🔴취학통지서(강도7.9)·초등 입학준비 / 🔴크리스마스·호두까기인형 / 전집 | 출력 피크 |

핵심 통찰(검증): 쌍봉이 아니라 **① 겨울 입학 대산맥(12→1→2월, 실측 피크 1월) ② 3월 새학기 봉 ③ 7~8월 여름 트래픽 피크 ④ 9→11월 유치원 입학(처음학교로) 산맥**. 나들이 표적은 bare "가볼만한곳"이 아니라 **"아이랑/어린이" 명시어**만 우리 유저([표적 리스트](DEMAND_TARGETS.md)). 발행은 각 윈도우를 *몇 주 앞서* 비워둘 것.

## seed 키워드 세트
- 머신리더블: [`packages/db/scripts/demand/seed-keywords.ts`](../packages/db/scripts/demand/seed-keywords.ts) — 8카테고리 ~50개, 부모 실제 검색어 기준. 각 seed에 `hypothesisMonths`·`painkiller` 가설 태깅(위 캘린더와 1:1).
- Step 2 진단 스크립트가 이 배열을 그대로 API에 투입.

## 4단계 플랜 (현재 = Step 2 완료, 2026-06-08)

- **[x] Step 0 — seed 세트 + 시즌 캘린더 가설** (이 문서 + seed 파일).
- **[x] Step 1 — API 키 발급·활성** (검색광고 3종 + 데이터랩 2종, `.env`/`.env.example` 반영).
- **[x] Step 2 — 진단·조인·검증** (`diagnose`·`magnitude`·`join`·`niche`·`validate-demo`·`dashboard`). 산출: 위 시즌 캘린더 *확정판* + 정렬 격차([DEMAND_FINDINGS](DEMAND_FINDINGS.md)·[JOIN](DEMAND_JOIN.md)) + 표적 리스트([DEMAND_TARGETS](DEMAND_TARGETS.md)) + 대시보드(demand-dashboard.html). seed 교정 반영.
- **[ ] Step 3 — 시즌 캘린더 → 키데이트 배너·발행 우선순위 배선**. 첫 사용자 가치(시점성). 첫 실탄 = 9~11월 처음학교로 입학 D-day.
- **[ ] Step 4 — 주간 수요 잡 + 리서치 에이전트 주입** (가치 증명 후). 전담 "트렌드 스카우트" 승격은 트리거 충족 시 별도 판단.

## 전담 에이전트?
지금은 **아니오**. 키워드/검색량은 결정론적 API+크런칭이라 LLM 불필요. LLM 가치는 *해석*(키워드 군집 → 8카테고리/5타입 매핑 + "왜 공명") 뿐 → 기존 리서치(스카우트) 에이전트 프롬프트에 "이번 주 수요 신호" 입력으로 주입. 독립 직군 승격은 (a)코퍼스 비대화 (b)별도 주기(주간) (c)큐레이션 적중률 측정 향상이 보일 때.
